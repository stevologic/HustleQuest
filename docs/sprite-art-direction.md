# Sprite Art Direction (HustleQuest)

This document translates the provided sprite references into implementation rules for UI and game-feel progression.

## Visual pillars

1. **Pixel identity**: use chunky icon silhouettes, high-contrast borders, and compact labels.
2. **Class fantasy**: services are presented like playable classes (Tutor, Designer, Gardener, etc.).
3. **Progress clarity**: trust/speed/quality stats are always visible and never framed as guarantees.

## Icon-to-feature mapping

- Shield/check icons → verification and trust indicators.
- Lightning/clock icons → response-time and reliability.
- Star/trophy icons → quality and earned achievements.
- Coin/gift icons → non-cash progression rewards.
- Lock/key icons → tier-gated features (Private vs Business).

## UI usage

- Home hero: “service RPG” framing + category chips.
- Listing cards: class tag, level badge, trust indicators.
- Dashboard: XP and milestone timeline.
- Admin: moderation reason badges with clear labels.

## Compliance guardrails

- Badge text must not imply guaranteed safety, quality, or earnings.
- Reviews must indicate verified-contact logic explicitly.
- Rewards cannot be random paid chance mechanics.

## Asset handling

- Treat uploaded sprites as inspiration only unless ownership/license is confirmed.
- Keep a provenance log for any production asset pack.
